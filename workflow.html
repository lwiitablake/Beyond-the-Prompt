Responsible AI Workflow

A repeatable process for making thoughtful instructional decisions about AI.

Use this workflow when revising course materials, creating examples, reviewing AI output, or deciding whether AI belongs in a task at all.

Why this workflow scales

What should you do? Use the same decision process across tools, disciplines, and learning contexts.

Why is this better for learners? A stable process helps keep privacy, accessibility, equity, and learning at the center even when AI tools change.

The seven steps

Open each step as needed, or expand everything to see the full workflow.

Step 1 Identify the instructional problem What are you trying to improve?

What should you do?

Name the specific teaching or learning problem before choosing an AI tool.

Why is this better for learners?

A clear problem keeps the work focused on learning instead of technology. It also helps you avoid using AI when a simpler fix would work better.

Common situations

  • Directions are confusing or too long.
  • Examples feel generic or disconnected.
  • Materials may have access barriers.
  • You need a faster way to organize rough ideas.

Common mistakes

  • Starting with the tool instead of the problem.
  • Trying to fix too many things at once.
  • Defining the task so broadly that the output becomes generic.

Try it

Write one sentence: “I want to improve ______ so learners can ______.”

Step 2 Decide if AI is appropriate Should AI help with this task?

What should you do?

Pause before using AI. Check whether the task is safe, useful, and appropriate for AI support.

Why is this better for learners?

Choosing when not to use AI protects privacy, preserves human judgment, and keeps learning from being outsourced.

Check before continuing

  • Private information: Does this include names, grades, IDs, messages, accommodations, or submissions?
  • High-stakes decisions: Does this affect grades, eligibility, discipline, or access?
  • Learning value: Would AI replace the skill learners are supposed to practice?
  • Policy: Does your institution allow this use?

Try it

Write either “AI is appropriate here because…” or “AI is not appropriate here because…”.

Step 3 Gather context What information does the work need?

What should you do?

Collect the learning goal, learner needs, task purpose, constraints, and review criteria before drafting.

Why is this better for learners?

Context helps the draft fit the actual course instead of sounding generic, vague, or disconnected.

Helpful context

  • Learning outcome or goal
  • Audience and prior knowledge
  • Reading level and language expectations
  • Accessibility requirements
  • LMS or course format constraints
  • Assignment purpose and grading expectations

Try it

List three pieces of context AI would need to produce a useful draft.

Step 4 Generate a draft How can AI create a starting point?

What should you do?

Use AI to create a draft, outline, revision, checklist, example, or alternative wording.

Why is this better for learners?

A draft can save time, but only if you use that time to improve clarity, access, relevance, and support.

Common mistakes

  • Treating the first output as finished.
  • Letting AI change the task without noticing.
  • Keeping polished wording that is vague or inaccurate.
  • Using examples that do not reflect your learners or context.

Try it

Ask for a draft, then immediately ask: “What would a learner still find confusing?”

Step 5 Use human review and judgment What must you check before using it?

What should you do?

Review the draft for accuracy, accessibility, privacy, alignment, plain language, inclusion, and learner support.

Why is this better for learners?

Human review helps prevent confident errors, hidden barriers, stereotypes, and confusing instructions from reaching learners.

Review dimensions

  • Accuracy: Are facts, examples, links, and citations correct?
  • Accessibility: Are headings, links, images, media, tables, and reading order usable?
  • Privacy: Is private learner or institutional information protected?
  • Alignment: Does the draft support the learning goal?
  • Plain language: Can the audience understand it quickly?
  • Learner impact: Could the tone, examples, or assumptions exclude or discourage some learners?

Try it

Mark one AI output as: use, revise, reject, or ask for another draft.

Step 6 Ask if it better supports learners Does this improve the learning experience?

What should you do?

Before sharing, ask whether the revised material is clearer, easier to use, more accessible, and better aligned with the learning goal.

Why is this better for learners?

Finished is not the same as useful. This step keeps the focus on learner success, not task completion.

Check before sharing

  • Can learners tell what to do first?
  • Are expectations clear?
  • Are there unnecessary barriers?
  • Are there meaningful options when appropriate?
  • Would this reduce confusion?

Try it

Finish this sentence: “This better supports learners because…”

Step 7 Reflect and improve What should change next time?

What should you do?

Review whether AI actually helped, what you changed, and what should be reused or revised next time.

Why is this better for learners?

Reflection turns one fix into a stronger, repeatable practice that can improve future learning experiences.

Reflection questions

  • Did AI save time without lowering quality?
  • Did the final material better support learners?
  • What needed the most human revision?
  • Should this become a template, example, or checklist?

Try it

Write one note for your future self: “Next time, I will…”
